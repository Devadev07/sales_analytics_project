# Power BI Dashboard — Build Guide

> ✅ **Status: built.** The finished dashboard is saved as
> [`sales_dashboard.pbix`](sales_dashboard.pbix) and a preview is in
> [`charts/powerbi_dashboard.png`](charts/powerbi_dashboard.png).
> This guide documents how it was built and how to reproduce it.

This guide walks you through building the dashboard on **your own machine** (Power BI
Desktop is Windows-only, so it can't run in this repo's environment). Everything it
needs is already prepared: just point Power BI at **`data/superstore_clean.csv`**.

By the end you'll have a 1-page interactive dashboard you can screenshot into
`charts/` and reference on your CV and LinkedIn.

---

## 0. Prerequisites
- **Power BI Desktop** installed (free from the Microsoft Store).
- The file `data/superstore_clean.csv` (already generated by `python src/clean_data.py`).

---

## 1. Load the clean data
1. Open Power BI Desktop → **Home → Get Data → Text/CSV**.
2. Select `data/superstore_clean.csv` → **Load**.
3. In the **Data** view, confirm types:
   - `Order Date`, `Ship Date` → **Date**
   - `Sales`, `Profit`, `Discount`, `Profit Margin` → **Decimal number**
   - `Quantity`, `Order Year`, `Order Month` → **Whole number**

> The CSV is already cleaned (no nulls/dupes, dates parsed), so you can go straight
> to building visuals — no Power Query cleanup needed.

---

## 2. Create 3 measures (Modeling → New measure)
Paste each one separately:

```DAX
Total Revenue = SUM(superstore_clean[Sales])
```
```DAX
Total Profit = SUM(superstore_clean[Profit])
```
```DAX
Profit Margin % = DIVIDE([Total Profit], [Total Revenue])
```
Format `Profit Margin %` as a percentage (Measure tools → % button).

---

## 3. Build the visuals (target layout below)

```
+------------------+------------------+------------------+
|  KPI: Revenue    |  KPI: Profit     |  KPI: Margin %   |   <- Card visuals
+------------------+------------------+------------------+
|     Monthly revenue trend (line chart)                 |
+--------------------------------+-----------------------+
|  Top 10 products (bar)         |  Sales by region (map |
|                                |  or donut)            |
+--------------------------------+-----------------------+
```

### a) KPI cards (top row)
- Insert **Card** visual ×3.
- Drag `Total Revenue`, `Total Profit`, and `Profit Margin %` — one per card.
- Expected values: **Revenue ≈ $2.30M, Profit ≈ $286K, Margin ≈ 12.5%**.

### b) Monthly revenue trend (line chart)
- Visual: **Line chart**.
- X-axis: `Order Date` (set to Date hierarchy → Month, or use `Order Month Name`).
- Y-axis: `Total Revenue`.
- Optional: add `Total Profit` as a second line.

### c) Top 10 products (bar chart)
- Visual: **Clustered bar chart**.
- Y-axis: `Product Name`  •  X-axis: `Total Revenue`.
- Filter pane → `Product Name` → **Top N → Top 10 by Total Revenue**.

### d) Sales by region (map or donut)
- Visual: **Donut chart** (simplest) — Legend: `Region`, Values: `Total Revenue`.
- Or **Filled map** — Location: `State`, Color saturation: `Total Revenue`.
- Expected leader: **West ≈ $725K**.

---

## 4. Add a slicer (interactivity = interview talking point)
- Insert a **Slicer** visual → field `Category` (or `Region`).
- Now clicking a category filters the whole page. This "cross-filtering" is the
  thing interviewers love to see in a Power BI demo.

---

## 5. Polish & export
1. Give the page a title text box: *"Superstore Sales Dashboard (2015–2018)"*.
2. **File → Save As** → `sales_dashboard.pbix` in the repo root.
3. **File → Export → Export to PDF** (optional, for the CV portfolio).
4. Take a screenshot of the full page → save as `charts/powerbi_dashboard.png`.
5. Commit `sales_dashboard.pbix` and the screenshot, then push.
